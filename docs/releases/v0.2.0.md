# AdofaiIpc v0.2.0

AdofaiIpc를 의존성으로 사용하는 모드가 IPC를 자동으로 설치할 수 있도록 공용 Bootstrap을
추가한 릴리스입니다.

의존 모드는 작은 `AdofaiIpc.Bootstrap.dll`을 함께 배포합니다. 게임에 AdofaiIpc가 없으면
Bootstrap이 GitHub Releases에서 최신 패키지를 내려받아 설치하고, IPC를 활성화한 다음 원래
모드의 진입점을 실행합니다.

## 주요 변경사항

- 재사용 가능한 `AdofaiIpc.Bootstrap.dll` 추가
- 의존 모드별 본체 DLL과 진입점을 지정하는 Bootstrap manifest 지원
- GitHub Releases의 최신 `AdofaiIpc.zip` 자동 다운로드
- `AdofaiIpc.zip.sha256`을 이용한 SHA-256 무결성 검증
- ZIP path traversal을 차단하는 안전한 압축 해제
- 임시 폴더에 먼저 설치한 뒤 `Mods/AdofaiIpc`로 이동하는 설치 흐름
- 여러 의존 모드가 동시에 시작할 때 중복 설치를 피하는 설치 작업 공유
- 설치 후 Unity main thread에서 AdofaiIpc와 의존 모드 활성화
- Windows, macOS, Linux에서 같은 Bootstrap 패키지 사용
- VS Code와 일반 MSBuild가 두 프로젝트의 게임 assembly 경로를 공유하도록 설정 정리

## 사용자

Bootstrap을 포함한 모드를 설치한 경우 AdofaiIpc를 따로 설치하지 않아도 됩니다. AdofaiIpc가
없으면 모드 로딩 중 자동으로 설치됩니다.

AdofaiIpc가 이미 설치되어 있지만 비활성화된 경우에는 사용자 설정을 임의로 변경하지 않습니다.
UnityModManager에서 AdofaiIpc를 직접 활성화해야 합니다.

수동 설치 방식도 그대로 지원합니다.

1. 릴리스에 첨부된 `AdofaiIpc.zip`을 다운로드합니다.
2. 압축 파일 안의 `AdofaiIpc` 폴더를 ADOFAI의 `Mods` 폴더에 넣습니다.
3. ADOFAI를 실행하거나 다시 시작합니다.

## 모드 개발자

의존 모드 패키지에 다음 파일을 포함합니다.

```text
ExampleMod/
├── AdofaiIpc.Bootstrap.dll
├── AdofaiIpcBootstrap.json
├── ExampleMod.dll
└── Info.json
```

`Info.json`의 `AssemblyName`과 `EntryMethod`는 Bootstrap을 가리키고,
`AdofaiIpcBootstrap.json`에는 실제 모드 DLL, 진입점, 최소 AdofaiIpc 버전과 릴리스 URL을
지정합니다.

Bootstrap 자체는 `AdofaiIpc.dll`을 참조하지 않으므로 IPC가 설치되지 않은 상태에서도 먼저
로드될 수 있습니다.

## 다운로드 검증

릴리스에 함께 첨부된 체크섬으로 ZIP을 검증할 수 있습니다.

```bash
shasum -a 256 -c AdofaiIpc.zip.sha256
```
