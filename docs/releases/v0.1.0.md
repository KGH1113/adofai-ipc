# AdofaiIpc v0.1.0

AdofaiIpc의 첫 공개 릴리스입니다.

AdofaiIpc는 A Dance of Fire and Ice 모드들이 브라우저, userscript, CLI 같은 외부
클라이언트와 통신할 수 있도록 하나의 localhost HTTP listener를 제공하는 IPC 모드입니다.
각 모드는 별도의 서버를 열지 않고 자신만의 namespace와 method를 등록할 수 있습니다.

## 주요 기능

- 여러 ADOFAI 모드가 공유하는 단일 HTTP listener
- 모드별 namespace와 method 등록 및 해제
- namespace와 method discovery endpoint
- Unity main thread 작업을 위한 `RegisterMainThread`
- namespace별 허용 origin 설정
- 기본 포트 `32145`와 `32155`까지의 자동 fallback
- JavaScript와 TypeScript를 지원하는 `@adofai-ipc/client`
- UnityModManager 직접 로딩

AdofaiIpc는 독립적으로 동작하며 JALib을 요구하지 않습니다.

## 설치

1. 릴리스에 첨부된 `AdofaiIpc.zip`을 다운로드합니다.
2. 압축 파일 안의 `AdofaiIpc` 폴더를 ADOFAI의 `Mods` 폴더에 넣습니다.
3. ADOFAI를 실행하거나 다시 시작합니다.

설치 결과는 다음 구조가 되어야 합니다.

```text
Mods/
└── AdofaiIpc/
    ├── AdofaiIpc.dll
    └── Info.json
```

필요한 런타임 의존성은 UnityModManager뿐입니다.

## 동작 확인

ADOFAI 실행 후 다음 요청으로 listener 상태를 확인할 수 있습니다.

```bash
curl -s http://127.0.0.1:32145/ipc/health
```

`32145`가 이미 사용 중이면 AdofaiIpc는 `32146`부터 `32155` 사이의 사용 가능한 포트를
선택합니다. 이 경우 클라이언트는 `/ipc/health`를 순서대로 호출해 실제 포트를 찾을 수
있습니다.

## 모드 개발자

모드는 `AdofaiIpc.dll`을 참조하고 public API를 통해 namespace를 등록합니다.

```csharp
var ipc = AdofaiIpc.AdofaiIpc.RegisterNamespace(
    "example-mod",
    new AdofaiIpc.IpcNamespaceInfo {
        DisplayName = "Example Mod",
        Version = "1.0.0"
    }
);

ipc.Register("health", _ => new {
    ok = true
});
```

상세 사용법은 [개발 가이드](../kor/000-DevelopGuide.md)와
[JavaScript/TypeScript Client 문서](../kor/005-JsClient.md)를 참고하세요.

## 다운로드 검증

릴리스에 함께 첨부된 `AdofaiIpc.zip.sha256` 파일로 다운로드한 ZIP의 무결성을 확인할 수
있습니다.

```bash
shasum -a 256 -c AdofaiIpc.zip.sha256
```
